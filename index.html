Data

The model turns ordinary cafeteria counts into operational timing.

No student names. No cameras. No private records. Just aggregate cafeteria patterns and a narrow question: what will return unopened, and what can still be safely rescued?

Architecture

01 / Feature Intake

Aggregate cafeteria signals become a structured feature vector.

Twenty-two features describe the operating day: menu, period, attendance, weather, event, line placement, cooler capacity, monitor status, and after-school demand.

02 / Prediction Heads

Specialized models score the same day from different angles.

A risk classifier, quantile return forecasters, demand regressor, and anomaly detector run together so the demo shows uncertainty instead of one brittle number.

03 / Safety Compiler

Predictions are compiled into allowed and blocked actions.

The rescue-window state machine checks packaging, monitor status, cooler capacity, cold-chain assumptions, and policy approval before any action can count.

04 / Human Loop

The output is not a chart. It is an approval-ready decision.

Second Bell explains the action, deadline, safety gate, confidence range, labor minutes, and impact receipt. Ms. Rivera still approves or blocks it.
